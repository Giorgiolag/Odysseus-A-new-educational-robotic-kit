## Hardware Architecture

The hardware architecture of the Odysseus (Ody) platform is designed around a custom, two-layer printed circuit board (PCB) that acts as a comprehensive shield for an Arduino Uno R3 core. The system is engineered to prioritize modularity, safety for young users, and uninterrupted operation in classroom environments.

### Microcontroller Subsystem
The processing architecture utilizes a dual-microcontroller approach:
* **Main Controller (ATmega328P):** An Arduino Uno R3 clone serves as the primary processing unit. It hosts the customized Firmata protocol, parsing high-level commands (received via USB or Bluetooth) and executing the corresponding hardware-level actions (e.g., motor PWM, I2C sensor reading).
* **Auxiliary Co-processor (ATtiny85):** To offload specific peripheral tasks and conserve memory on the primary controller, an ATtiny85 microcontroller is integrated directly onto the custom PCB via a DIP-8 socket. Its dedicated function is to drive the OLED display. It operates as an I2C slave to communicate with the ATmega328P, while simultaneously acting as an I2C master to the OLED display on a separate channel. All available bitmap images are stored locally in the ATtiny85's memory. When an image needs to be displayed, the main controller simply transmits the corresponding image index, and the ATtiny85 handles the full fetching and rendering process independently.

### Power Management
The system is powered by two 18650 Lithium-Ion batteries wired in series, providing a nominal 7.4V supply. To safely distribute this power across various logic and high-current components, the power management circuit incorporates:
* **Dual Linear Regulation:** Two separate L7805CD2T-TR (5V) linear voltage regulators are utilized. This topology isolates the logic circuitry from the electrical noise and voltage drops generated by the physical actuators (DC motors and servos).
* **Protection Circuitry:** An IRF4905S P-Channel Power MOSFET is integrated to provide robust reverse-polarity protection, safeguarding the sensitive electronics from improperly inserted batteries.

### Actuator and Sensor Interfaces
Odysseus is designed to be fully modular, providing numerous plug-and-play interfaces.
* **Motor Control:** A dedicated MX1508 motor driver IC provides bi-directional control for up to two DC motors. Continuous rotation and standard angle servos are driven directly via the Arduino's PWM-capable pins.
* **Physical Safety and Orientation:** A key design constraint for educational hardware is mitigating user error. All sensor and servo plug housings are physically constrained by the 3D-printed enclosure to enforce a specific orientation, preventing short circuits via incorrect insertion. For the DC motor and LED sockets, the connections are non-destructive; reverse insertion merely results in reversed motor rotation or an unlit LED.

### Communication and Logic
Wireless communication is facilitated by a JDY-31 Bluetooth UART module (2.4GHz). A common issue in Arduino-based robotics is the conflict between the Bluetooth module and the USB serial line during firmware uploads. To resolve this, the architecture employs an **SN74LVC2G66 Analog Switch**. This logic switch seamlessly isolates the Bluetooth module's TX/RX lines during programming, eliminating the need for users to physically unplug the Bluetooth module when connecting the robot to a PC.
